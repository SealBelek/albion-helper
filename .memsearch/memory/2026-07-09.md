## Plan: Background API Enricher

### Problem
Cities like Bridgewatch, Caerleon, and Black Market have 0 opportunity groups because the NATS stream provides only one-sided order data for each item. The opportunity query requires **both** bid and ask orders for the same `(item_id, city, quality)` group.

### Solution
A background goroutine that periodically fetches prices for many items from the Albion Data API and populates both order sides into `marketorders`.

### Architecture

**New file: `internal/enricher/enricher.go`**
- `Enricher` struct holding `*sql.DB` and `*api.Client`
- `Start()` goroutine:
  1. Queries `SELECT UniqueName FROM items` filtered to T4-T8 (~9,300 items)
  2. Shuffles randomly each cycle
  3. Batches items into URL-friendly groups (~195 items/request, fits 4000-char limit)
  4. Calls `api/stats/prices/{items}.json?qualities=0,1`
  5. Inserts via existing `insertPriceSnapshot()` (deletes old `source='api'` rows, inserts both offer/request sides)
  6. Full cycle: ~48 requests × ~0.5s = ~24s per cycle, repeats infinitely

**New file: `internal/api/ratelimit.go`**
- Token bucket rate limiter: **2 tokens/sec, burst 4** (120 req/min, well under 180/min limit)
- Shared between enricher and existing `FetchPrices` calls
- `Wait()` method blocks until a token is available

**Modified: `internal/api/client.go`**
